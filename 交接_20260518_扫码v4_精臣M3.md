## 关键技术决策(背后 why)

### 1. 多库梯度 vs 一上来就 AI

按 plan `BarcodeDetector → ZXing 0.21 → jsQR+Quagga2 → AI` 顺序探索。**iOS Safari 没 BarcodeDetector**(Chromium 独占,这点查清了),fall back 路径正确。jsQR + Quagga2 + ZXing 三路并行框架 + setInterval 保留,任一命中即停,诊断条标记 `[jsQR]/[ZXing]/[Quagga]` 看赢家。

### 2. quiet zone 是真扫不出的根因(PR #20)

之前 PR #12 把 `JsBarcode margin:0` 当「省空间技巧」。**CODE128 标准要求两侧 quiet zone**,无 quiet zone 时扫描器找不到 start/stop pattern 直接放弃。第三方 barcode 都有 quiet zone 所以一发入魂,自家的没有所以死活扫不出。这是看起来玄学实际标准明文的失败模式。

### 3. 砍 SKU 编码长度才是小标签的真解(PR #24)

PR #21/#23 我把渲染分辨率从 `width:4` 拉到 `width:24`(3500×660 px 源,等效 2470 DPI)。**用户屏幕扫屏幕**场景下,源分辨率已经不是瓶颈——**编码内容长度才是**。`tw20260427020735407092`(23 字符自动 ID)→ 290 modules,36mm 屏宽下每 module 仅 0.124mm,手机摄像头隔 10-15cm 拍每 module 仅 ~1.3 capture 像素物理上 borderline。

解法:`getBarcodeContent(p)`,SKU ≤12 用 SKU,否则用 `p.id.slice(-8)`(随机/timestamp 尾,前缀 `tw` + date 解耦)。`showScanResult` 加 endsWith 兜底,旧标签全长 ID 仍精确命中。

### 4. 矢量 PDF 替代 PNG(PR #24)

`drawBarcodeVector(pdf, text, x, y, w, h)`: JsBarcode 生成 SVG → 解析 `<g translate>` + `<rect>` → `pdf.rect()` 一条条画。PDF 内 barcode 是真矢量,viewer 缩放/打印任意 DPI 都锐利,告别 PNG 中间桥的抗锯齿/插值。HTML 预览仍用 PNG(浏览器 `<img>` 显示足够好)。

### 5. 精臣 M3 单页 PDF(PR #25)

精臣 M3 是**300 DPI 热转印,印宽 20-80mm**(用户纠正了我假设的 203 DPI;`large 90×60` 和 `big 120×80` 超规格根本印不出来)。云印 app 需要每页 = 标签尺寸的多页 PDF,直接连续打印。

`label-pdf-mode` select:`single`(默认,精臣)/ `grid`(A4 网格家用)。Single 模式 `format=[cfg.w, cfg.h]` + 显式 `orientation`(jsPDF 默认 portrait 会反转 40×30 把 barcode 印成竖的)+ 不画虚线外框(页面 IS 标签)。
