# 矿珍库 · 多币种支持 交接 (2026-05-17)

> 本次会话围绕多币种价格展开 — schema 加 `currency` 列、UI 加 select 切换、汇率 API 接入。重点是数据语义,跑 SQL 之前先看完。

## 一、最重要 — 必须先跑的 SQL

⚠️ **跑之前先备份(在 Supabase Dashboard → Database → Backups,或手动跑 `CREATE TABLE products_backup AS SELECT * FROM products;`)**

```sql
-- 加 currency 列
ALTER TABLE products ADD COLUMN IF NOT EXISTS currency TEXT;
ALTER TABLE logs ADD COLUMN IF NOT EXISTS currency TEXT;

-- 现有数据按 CNY 标记(实际金额数值就是 CNY,不要动数值)
UPDATE products SET currency='CNY' WHERE currency IS NULL;
UPDATE logs SET currency='CNY' WHERE currency IS NULL;

-- 默认值改 JPY(只影响未来新插入的行)
ALTER TABLE products ALTER COLUMN currency SET DEFAULT 'JPY';
ALTER TABLE logs ALTER COLUMN currency SET DEFAULT 'JPY';

-- 验证
SELECT 'products' AS tbl, currency, COUNT(*) FROM products GROUP BY currency
UNION ALL
SELECT 'logs', currency, COUNT(*) FROM logs GROUP BY currency;
```

**为什么旧数据标 CNY 而不是 JPY**:跑过更早一条 SQL `UPDATE products SET price=price*3` 之后,`products.price` 的数值是 "CNY 进价 × 3 的数字"。如果直接改 currency 标 JPY,系统会把 `390` 当 ¥390 日元来显示和换算,实际是 ¥390 人民币(约 ¥7,995 JPY)。用户已经被这个坑过一次。

## 二、数据语义约定

| 表/字段 | 含义 |
|---|---|
| `products.price` + `products.currency` | 售价 + 售价的币种 |
| `logs.price` + `logs.currency` | 当次入库进价 / 出库售价 + 币种(实际入库填的) |
| 推荐价格(动态算) | 该商品所有 `type='in'` 的 `logs.price` 各自换算到 JPY → 平均 → ×3,**固定基于 JPY 计算**,显示时再按 `detailCurrency` 换算 |

## 三、UI 状态

**两个独立的"显示货币"**(各自 localStorage,默认 JPY):
- `inventoryCurrency` (key `mz_inv_currency`) — 库存列表显示
- `detailCurrency` (key `mz_detail_currency`) — 商品详情 modal 显示

两者互不影响,通过 `setInventoryCurrency()` / `setDetailCurrency()` 切换,各自只触发对应页面重渲染。

**3 个 modal 里的"输入币种"** select 默认全 JPY:
- `f-currency` (建品/编辑商品)
- `si-currency` (入库 modal)
- `out-currency` (出库 tab)

## 七、几次踩坑记录

1. **商品全显示「已删除」** — 当时把 `currency` 加进 products select 字段列表,但用户还没跑 ALTER。PostgREST 直接 400,`DB.products=[]`,流水里 `getProduct()` 全 null。修复:loadAll 加 fallback select(不带 currency),upsertProduct/insertLog 也加 fallback。
2. **币种数值失真担忧** — 用户敏锐发现"只改 currency 标记不动数值会出大错"。最终保留旧数据 CNY 标记,只改默认值。
3. **推荐价格不跟币种变** — 一开始按设计写死 JPY 显示,后改成基于 JPY 计算 + fmtPrice 按 detailCurrency 显示。
